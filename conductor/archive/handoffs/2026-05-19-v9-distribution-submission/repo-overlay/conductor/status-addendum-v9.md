# Conductor status addendum v9

## Umbrella track

`anz-platform-release-and-distribution` should become the umbrella delivery track for:

- release reconciliation
- ANZ provider hardening
- capability manifest
- MCP/export routing
- npm and website publishing
- MCP registry submissions
- Claude/Codex/Copilot integrations
- VS Code/OpenVSX extension planning
- Gemini/Qwen tool routes
- Rust migration readiness

## Merge guidance

Do not delete existing tracks automatically.

Classify existing tracks as:

- active subtrack
- completed input
- backlog input
- archived historical input

Then update `conductor/status.md` so there is no contradiction between active-track inventory and the umbrella track.
