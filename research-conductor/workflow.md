# Research Workflow

**Programme:** NZ Health Legislation Corpus Programme  
**Version:** 1.0  
**Date:** 2026-03-11  
**Status:** ⚠️ DRAFT - Requires ethics approval

## Overview

This document defines the 7-phase research workflow with stage gates for the NZ Health Legislation Corpus Programme.

**Duration:** 10-12 weeks per track  
**Parallel Execution:** Up to 5 tracks simultaneously  
**Output:** NZMJ manuscript per track

## Phase 1: Framing (Week 1)

### Objectives
- Refine research question
- Document differentiation from prior work
- Confirm election relevance
- Assess NZMJ fit
- Prepare OSF pre-registration draft

### Tasks
1. Document research question refinement
2. Document differentiation from prior work (if applicable)
3. Document 2026 election relevance
4. Assess NZMJ fit (scope, audience, timing)
5. Identify data dependencies
6. Prepare OSF pre-registration draft
7. Create framing document for Gate 1 review

### Deliverables
- Framing document
- OSF pre-registration draft
- Gate 1 review checklist completed

### Gate 1: Framing Review

**Criteria:**
- [ ] Research question clearly stated
- [ ] Differentiation from prior work documented (if applicable)
- [ ] Election relevance established
- [ ] NZMJ fit confirmed
- [ ] Data dependencies identified
- [ ] OSF pre-registration draft ready

**Decision:** Proceed to Phase 2 / Revise / Hold

---

## Phase 2: Corpus (Weeks 2-3)

### Objectives
- Harvest legislation from NZ Legislation API
- Build metadata tables
- Document corpus boundaries
- Run sensitivity analysis (alternative corpus definitions)

### Tasks
1. Define inclusion/exclusion rules
2. Harvest works from NZ Legislation API
3. Retrieve version history for each work
4. Download XML for available versions
5. Build metadata table (work_id, version, dates, status)
6. Document corpus boundaries
7. Create corpus summary statistics
8. Run sensitivity analysis (alternative corpus definitions)

### Deliverables
- Corpus metadata table
- XML archive
- Sensitivity analysis notes
- Gate 2 review checklist completed

### Gate 2: Corpus Review

**Criteria:**
- [ ] Works list complete and dated
- [ ] Version list complete
- [ ] XML availability confirmed
- [ ] Metadata table generated
- [ ] Corpus boundaries documented
- [ ] Sensitivity analysis complete

**Decision:** Proceed to Phase 3 / Revise / Hold

---

## Phase 3: Coding (Weeks 4-5)

### Objectives
- Draft codebook
- Run pilot coding (20% for inter-rater reliability)
- Achieve κ > 0.7 for all categories
- Complete expert review
- Freeze coding rules

### Tasks
1. Draft initial codebook (section types, change types, domains, implementation indicators)
2. Select pilot sample (20% of corpus)
3. Train second coder (or self-code with time separation)
4. Run pilot coding
5. Calculate inter-rater reliability (Cohen's kappa)
6. Target κ > 0.7 for all categories
7. Resolve coding disagreements
8. Revise codebook
9. Run expert review (5-10% sample)
10. Freeze coding rules for main run

### Deliverables
- Frozen codebook
- Pilot results
- Inter-rater reliability report (κ > 0.7)
- Expert review feedback
- Gate 3 review checklist completed

### Gate 3: Coding Review

**Criteria:**
- [ ] Codebook drafted with all categories defined
- [ ] Pilot sample coded (20%)
- [ ] Inter-rater reliability κ > 0.7 for all categories
- [ ] Disagreements documented and resolved
- [ ] Expert review completed
- [ ] Coding rules frozen

**Decision:** Proceed to Phase 4 / Revise / Hold

---

## Phase 4: Analysis (Weeks 6-7)

### Objectives
- Run full coding on corpus
- Calculate primary metrics
- Run sensitivity analysis
- Attempt triangulation
- Draft tables and figures

### Tasks
1. Run full coding on corpus
2. Calculate tempo metrics (amendment rate, change velocity, reform density)
3. Compute pattern metrics (change type distribution, section targeting, clustering)
4. Calculate implementation burden metrics (if applicable)
5. Generate composite indices (volatility, burden, etc.)
6. Run sensitivity analysis (alternative corpus, time periods, weightings)
7. Attempt triangulation with policy documents
8. Draft tables (corpus characteristics, metrics)
9. Draft figures (time series, reform cycles)
10. Document limitations and uncertainty

### Deliverables
- Analysis results
- Sensitivity analysis report
- Draft tables
- Draft figures
- Gate 4 review checklist completed

### Gate 4: Analysis Review

**Criteria:**
- [ ] Full coding complete
- [ ] Primary outcomes computed
- [ ] Sensitivity analysis complete
- [ ] Triangulation attempted
- [ ] Tables drafted
- [ ] Figures drafted
- [ ] Limitations documented

**Decision:** Proceed to Phase 5 / Revise / Hold

---

## Phase 5: Drafting (Weeks 8-9)

### Objectives
- Write structured abstract
- Create argument outline
- Draft full manuscript
- Compile supplementary appendix
- Complete co-author review

### Tasks
1. Write structured abstract skeleton (Aims, Methods, Results, Conclusions)
2. Create argument outline
3. Draft Introduction (policy context, differentiation from prior work)
4. Draft Methods (corpus, coding, metrics, sensitivity)
5. Draft Results (tempo patterns, implementation burden, sensitivity)
6. Draft Discussion (implications, limitations, comparison to prior work)
7. Draft Conclusion (recommendations)
8. Compile supplementary appendix (codebook, data, scripts)
9. Internal review and revision
10. Co-author review

### Deliverables
- First full draft
- Supplementary appendix draft
- Co-author feedback
- Gate 5 review checklist completed

### Gate 5: Drafting Review

**Criteria:**
- [ ] Structured abstract drafted (200 words max)
- [ ] Argument outline complete
- [ ] Full manuscript drafted (3,000 words max)
- [ ] Supplementary appendix compiled
- [ ] Co-author review completed
- [ ] Word count within limit (3,000)
- [ ] Reference count within limit (30)

**Decision:** Proceed to Phase 6 / Revise / Hold

---

## Phase 6: Revision (Weeks 10-11)

### Objectives
- Humanizer pass (remove AI artifacts)
- NZMJ fit pass (formatting, style)
- Reviewer simulation (hostile reviewer #2)
- Complete OSF pre-registration
- Final co-author approval

### Tasks
1. Humanizer pass (remove AI artifacts, strengthen flow)
2. NZMJ compression pass (word count, style alignment)
3. Hostile reviewer simulation (identify weaknesses)
4. Address reviewer concerns
5. Final style pass (references, formatting)
6. Complete OSF pre-registration
7. Co-author review
8. Final revisions
9. ICMJE disclosure forms (all authors)

### Deliverables
- Revision-complete manuscript
- OSF registration complete
- ICMJE disclosure forms
- Gate 6 review checklist completed

### Gate 6: Revision Review

**Criteria:**
- [ ] Humanizer pass complete
- [ ] NZMJ fit pass complete (word count, references, abstract format)
- [ ] Reviewer simulation completed
- [ ] Reviewer concerns addressed
- [ ] OSF pre-registration complete
- [ ] Co-author approval obtained
- [ ] ICMJE forms collected

**Decision:** Proceed to Phase 7 / Revise / Hold

---

## Phase 7: Submission (Week 12)

### Objectives
- Format manuscript for NZMJ
- Prepare submission package
- Submit to NZMJ
- Prepare policy brief

### Tasks
1. Format manuscript as NZMJ docx
2. Prepare tables (publication-ready)
3. Prepare figures (publication-ready)
4. Finalize supplementary appendix
5. Write cover letter (with required statements)
6. Complete NZMJ submission forms
7. Submit to NZMJ (https://nzmj.manuscriptmanager.net)
8. Prepare policy brief (500 words)
9. Publish analysis code to GitHub (embargoed)
10. Prepare data for deposition (embargoed)

### Deliverables
- Submission complete
- Policy brief published
- Code repository ready (embargoed)
- Data repository ready (embargoed)
- Gate 7 review checklist completed

### Gate 7: Submission Complete

**Criteria:**
- [ ] Manuscript submitted to NZMJ
- [ ] Submission confirmation received
- [ ] Manuscript ID recorded
- [ ] Policy brief prepared
- [ ] Code repository ready (embargoed)
- [ ] Data deposition ready (embargoed)

**Decision:** Track complete / Monitor peer review

## Quality Assurance

### Inter-Rater Reliability
- 20% of corpus double-coded
- Target: κ > 0.7 for all categories
- Gate 3 checkpoint

### Sensitivity Analysis
- Alternative corpus boundaries
- Alternative time periods
- Alternative weightings
- Gate 4 checkpoint

### Expert Review
- 1-2 health policy experts
- 5-10% sample review
- Gate 3 checkpoint

### Co-Author Review
- At Gate 5 (drafting)
- At Gate 6 (revision)
- All authors approve submission
