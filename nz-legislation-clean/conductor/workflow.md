# Development Workflow

## Overview

This document defines the development workflow for the NZ Legislation Tool project.

---

## Branch Strategy

### Main Branch
- **Name:** `main`
- **Protection:** Protected branch
- **Deployments:** Auto-publish to npm on version tags

### Feature Branches
- **Naming:** `feature/<description>`, `fix/<description>`, `chore/<description>`
- **Source:** Branch from `main`
- **Merge:** Pull request with CI/CD passing

### Release Tags
- **Format:** `v<major>.<minor>.<patch>` (e.g., `v1.0.0`)
- **Trigger:** `release:` prefix in commit message
- **Process:**
  1. `npm version patch` (or minor/major)
  2. `git push --follow-tags`
  3. CI/CD publishes to npm

## Development Cycle

### 1. Setup
```bash
# Clone repository
git clone <repository-url>
cd nz-legislation-tool

# Install dependencies
npm install

# Set up environment
cp .env.example .env
# Edit .env with your API key
```

### 2. Development
```bash
# Run in development mode
npm run dev -- <command>

# Example
npm run dev -- search --query "health"
```

### 3. Testing
```bash
# Run tests in watch mode
npm test

# Run tests once
npm run test:run

# Run with coverage
npm run test:coverage
```

### 4. Code Quality
```bash
# Type check
npm run typecheck

# Lint (future)
npm run lint
```

### 5. Build
```bash
# Compile TypeScript
npm run build

# Output: dist/
```

## Commit Guidelines

### Commit Message Format
```
<type>: <description>

[optional body]

[optional footer]
```

### Types
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `test:` Test additions/changes
- `refactor:` Code refactoring (no behavior change)
- `chore:` Maintenance tasks
- `ci:` CI/CD changes
- `perf:` Performance improvements

### Examples
```bash
feat: Add rate limit configuration
fix: Handle API timeout errors gracefully
docs: Update README with new features
test: Add tests for CSV export
refactor: Simplify logger clearOldLogs method
chore: Update dependencies
ci: Add Node.js 22 to test matrix
```

### Release Commits
```bash
# Trigger release
git commit -m "release: v1.1.0"
```

---

## Pull Request Process

### 1. Create PR
- Branch from `main`
- Descriptive title
- Link to issues (if applicable)

### 2. PR Checklist
- [ ] Tests passing
- [ ] Type check passing
- [ ] Documentation updated
- [ ] Commit messages follow convention

### 3. Review
- At least 1 approval required
- Address all review comments
- Re-run CI/CD if needed

### 4. Merge
- Squash and merge (for feature branches)
- Delete branch after merge

## Release Process

### Version Numbering
Semantic Versioning: `MAJOR.MINOR.PATCH`

- **MAJOR:** Breaking changes
- **MINOR:** New features (backward compatible)
- **PATCH:** Bug fixes (backward compatible)

### Steps
```bash
# 1. Update version
npm version patch  # or minor/major

# 2. Push with tags
git push origin main --follow-tags

# 3. CI/CD automatically:
#    - Runs tests
#    - Builds dist/
#    - Publishes to npm
#    - Creates GitHub release
```

### Release Notes
Auto-generated by GitHub from:
- PR titles merged since last release
- Commit messages with `feat:`, `fix:`, etc.

### Common Issues

#### API Key Not Working
```bash
# Check configuration
nzlegislation config --show

# Re-set API key
nzlegislation config --key YOUR_API_KEY
```

#### Tests Failing
```bash
# Clear cache
rm -rf node_modules/.vite

# Re-run tests
npm run test:run
```

## CI/CD Pipeline

### Triggers
- **Push to `main`:** Full pipeline
- **Pull Request:** Lint + Test + Build
- **Version Tag:** Publish to npm

### Jobs
1. **Lint:** ESLint + TypeScript
2. **Test:** Vitest (Node 18, 20, 22)
3. **Build:** TypeScript compilation
4. **Publish:** npm publish (on version tag)

### Artifacts
- **dist/:** Compiled JavaScript (7 days retention)
- **coverage/:** Coverage reports (upload to Codecov)

## Troubleshooting

### Build Fails
```bash
# Clean and rebuild
rm -rf dist/ node_modules/
npm install
npm run build
```

### Tests Fail Randomly
```bash
# Check for race conditions
# Add explicit waits if needed
# Run tests sequentially: npm run test:run --no-parallel
```

### npm Publish Fails
```bash
# Check authentication
npm whoami

# Re-authenticate
npm login

# Check version (must be unique)
npm view nz-legislation-tool versions
```
