# ANZ Legislation Requirements and Contract Register

This is the single repository requirements register for provider, publication,
registry, integration, and future migration work. It uses MoSCoW priority to
separate release blockers from planned or future surfaces.

## MoSCoW policy

- **Must:** required before a stable release, external publication, registry
  submission, or public support claim.
- **Should:** required before promoting a prerelease surface or provider to the
  next maturity level.
- **Could:** useful expansion work that remains repository-local until its gates
  pass.
- **Won't now:** explicitly out of scope for the current delivery track.

## Validation contract

The repository must keep this file, `conductor/tracks.md`, and each
`conductor/tracks/<track-id>/` directory aligned. The
`gate:conductor-requirements` script checks that every required provider and
publication/registry track has:

- a row in this requirements register
- a track directory
- `metadata.json`
- `spec.md`
- `plan.md`
